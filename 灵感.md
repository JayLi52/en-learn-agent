## 灵感记录

### 2026-03-27

#### 文档向量化 RAG 系统

**需求确认：**
- 文档语言：中英文混合
- 文档规模：大型（100+文档 / GB级）
- 用途：RAG 知识库问答
- 用户环境：Windows 本地文件

**技术栈选型：**

**文档解析**：
- `python-docx` - 处理 DOCX 文件
- `python-pptx` - 处理 PPT/PPTX 文件
- `pdfplumber` 或 `PyMuPDF` - 处理 PDF 文件
- 注意：旧版 .doc/.ppt 格式可能需要 `antiword` 或 LibreOffice 转换

**文本分块**：
- LangChain `RecursiveCharacterTextSplitter` - 通用分块
- 考虑按段落、章节语义分块
- PPT 可按幻灯片分块，保留标题上下文

**向量数据库对比**（针对大规模场景）：

| 数据库 | 优点 | 缺点 | 适用场景 |
|--------|------|------|----------|
| Milvus | 开源、高性能、支持分布式 | 部署复杂度高 | 大规模生产环境 |
| Pinecone | 云托管、免运维、性能好 | 收费、数据在云端 | 不想管基础设施 |
| Chroma | 轻量、本地、开发简单 | 大规模性能有限 | 小型项目/原型 |
| Weaviate | 开源、支持混合搜索 | 资源占用较大 | 需要语义搜索的场景 |

**推荐方案**（针对大规模）：
- **首选**: Milvus（开源、可自托管、性能优秀）
- **备选**: Pinecone（不想管服务器时的选择）

**Embedding 模型选择**（中英文支持）：
- OpenAI `text-embedding-3-large` - 质量最高，中英文都好
- `BAAI/bge-m3` - 开源多语言模型，中英文效果不错，免费
- `sentence-transformers/paraphrase-multilingual` - 轻量多语言

**架构设计**：
```
Windows 文档夹
      ↓
  文档扫描器
      ↓
┌─────────────────────────────┐
│     文档解析管道            │
│  PDF → DOCX → PPTX → ...   │
└─────────────────────────────┘
      ↓
  文本分块器
      ↓
  Embedding 模型
      ↓
┌─────────────────────────────┐
│      向量数据库             │
│  Milvus / Pinecone          │
└─────────────────────────────┘
      ↓
  RAG 检索链路
      ↓
  LLM 生成回答
```
